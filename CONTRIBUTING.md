# Contributing

Thanks for helping make **Awesome Knowledge Infrastructure** better! This list is curated, so quality matters more than quantity.

## What belongs here

Tools, frameworks, and durable resources that help people or software **capture, connect, or operationalize knowledge** — personal knowledge management, documentation, developer portals, AI knowledge assistants, search/retrieval, knowledge graphs, decision records, and operational knowledge.

A good entry is:

- **Actively maintained** (or a genuinely canonical resource).
- **Broadly useful**, not a niche personal project.
- **Real and reachable** — the link must work (a link checker runs on every change).

## How to add an entry

1. Fork the repo and create a branch.
2. Add **one item per pull request** so each can be discussed on its own.
3. Use this exact format, placed in the most fitting section:

   ```markdown
   - [Name](https://example.com) - Short description that starts with a capital and ends with a period.
   ```

4. Keep descriptions to a single, specific sentence. Say what it *is* and what it's *for* — skip marketing adjectives.
5. Keep entries ordered sensibly within their section (alphabetical is fine).

## Suggesting a new section

Open an issue first so we can agree on scope before restructuring.

## Not sure?

Open an issue and ask. It's always fine to propose something and discuss.
